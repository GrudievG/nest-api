## Key Headers Explained

Content-Security-Policy     — restricts script/style/image/connect origins; 'none' for object-src
Cross-Origin-Opener-Policy  — isolates browsing context (mitigates Spectre)
Cross-Origin-Resource-Policy— prevents cross-origin reads of responses
Referrer-Policy             — no-referrer: URL not leaked in Referer header
Strict-Transport-Security   — HSTS, forces HTTPS for 1 year with subdomains
X-Content-Type-Options      — nosniff: prevents MIME-type sniffing
X-Frame-Options             — SAMEORIGIN: prevents clickjacking
X-XSS-Protection            — 0: disabled; modern browsers rely on CSP instead
X-RateLimit-Limit-global    — global policy bucket: 100 req/60s
X-RateLimit-Limit-strict    — strict policy bucket: 5 req/60s (on this endpoint)
x-request-id                — correlation ID for log tracing
Access-Control-Allow-Credentials — CORS credentials allowed (origin-restricted)
